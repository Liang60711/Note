## roles 需要區分環境，應該要另外建立 roles 還是使用同一個 roles 目錄?

在實務上，同一個 role 目錄就夠，不需要為每個環境再建立一份 role；除非是 windows 和 linux 的主機，才需要分開。

---

以下是一個 ansible 目錄結構，有區分環境。
```
ansible/
├── inventories/            ← 環境 inventory，存放變數
│   ├── dev/
│   │   ├── hosts
│   │   └── group_vars/web.yml
│   ├── staging/
│   │   ├── hosts
│   │   └── group_vars/web.yml
│   └── production/
│       ├── hosts
│       └── group_vars/web.yml
├── playbooks/              ← playbook 放這
│   ├── web.yml
│   └── db.yml
└── roles/
    └── nginx/              ← roles 放這
        ├── tasks/
        │   ├── main.yml    ← 每個 roles 的入口點
        │   ├── install.yml
        │   ├── config.yml
        │   └── service.yml
        ├── templates/
        │   └── nginx.conf.j2
        └── defaults/
            └── main.yml
```

<br/>

<br/>

- `playbooks/web.yml` : playbooks 可以一次呼叫許多 roles，每個 roles 的入口點是 `roles/xxx/tasks/main.yml`。

- playbook 一次可以呼叫多個 roles，並命名成 web.yml 或是 db.yml 之類的。
